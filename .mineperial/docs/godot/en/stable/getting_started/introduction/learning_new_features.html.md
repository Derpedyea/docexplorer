# Learning new features

*This section explains how to discover and understand new capabilities in Godot, and points you to the various learning resources available.*

## Making the most of this manual

- Use the sidebar navigation to jump to specific sections of the manual.
- Read the **Conceptual Documentation** first to understand the architecture before diving into APIs.
- Reference the **Built‑in Class Reference** for up‑to‑date method and property signatures.
- Search for keywords using the search bar at the top‑right.

## Learning to think like a programmer

- Start with the *“Introduction to Godot”* to grasp the overall workflow.
- Explore *“Key concepts”* such as **Scenes**, **Nodes**, and **Signals**.
- Practice by building small prototypes and iterating quickly.
- Keep a coding journal; write down patterns that work well for you.

## Learning with the community

- **Forums** – [Godot Q&A](https://godotengine.org/qa) for troubleshooting and idea exchange.
- **Discord** – Join the official Discord server for real‑time help.
- **GitHub** – Browse the repository for example projects and plugin contributions.
- **Reddit** – r/godot for community news and user projects.

## Community tutorials

- **Official Tutorials** – `tutorials/` directory in the docs, covering topics from *2D* to *3D* and *Scripting*.
- **YouTube Playlists** – Search “Godot tutorials” for visual walkthroughs.
- **Blogs** – Look for posts tagged `Godot` on platforms like Medium, Dev.to, or personal blogs.
- **Code‑share sites** – GitHub Gist, GitLab, or Bitbucket for reusable snippets.

> **Tip:** When following a tutorial, always check the *“What’s new in this version?”* section to adapt the steps to the latest Godot release.
