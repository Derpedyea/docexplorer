# Web

Godot can be exported to the web using the HTML5 export template.  
The web export uses WebAssembly to run the engine in the browser, and it
provides a bridge for communicating between the game and the surrounding
HTML/JavaScript environment.

---

## Overview

* **JavaScriptBridge singleton** – allows GDScript to call JavaScript and
  vice‑versa.
* **HTML5 shell class** – the class that implements the default browser
  interface (e.g. handling fullscreen, window resizing, etc.).
* **Custom HTML page** – you can supply your own index.html for the export,
  adding additional scripts or styling.

---

## 1.  Using the Web export template

1. **Install the template**  
   Download the *Web* export preset from the Export menu → “Install Export
   Templates” → “HTML5”.
2. **Create an export preset**  
   In *Project → Export*, click *Add…* → *HTML5*.  
   Configure the options (e.g. file name, window size, orientation).
3. **Export**  
   Click *Export Project*.  
   Godot will produce a directory with a *index.html*, *project.godot*,
   *web.zip*, etc.

---

## 2.  Browser support

| Browser | Support | Notes |
|---------|---------|-------|
| Chrome, Edge, Safari, Firefox | ✅ | Modern browsers that support WebAssembly and HTML5 APIs |
| IE11 | ❌ | Not supported – Godot requires WebGL 2.0+ |

---

## 3.  Limitations

| Feature | Status | Work‑around |
|---------|--------|-------------|
| File system access | ❌ | Use `FileAccess` with `FileAccess.READ` only |
| Networking (WebSocket) | ✅ | Use `WebSocketClient`/`WebSocketServer` |
| Audio playback | ✅ | Works with Web Audio API |
| Custom fonts | ✅ | Use @font-face in your HTML page |
| Fullscreen API | ✅ | Handled by the `HTML5Shell` |

## 4.  Custom HTML page for Web export

You can replace the default `index.html` with your own to embed the
game in a larger website or add custom scripts.

Example minimal `index.html`:

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>My Godot Game</title>
    <script src="godot.js"></script>
</head>
<body>
    <script>
        var engine = new Module({
            // Options for the exported module
            onRuntimeInitialized: function () {
                // Load the exported project
                Module.ccall('load_project', null, ['string'], ['project.godot']);
            }
        });
    </script>
</body>
</html>
```

Replace `godot.js` with the actual JavaScript file generated by the
export, and add any CSS or JS you need.

## 5.  The JavaScriptBridge singleton

The `JavaScriptBridge` singleton provides a simple API for communication.

```gdscript
# Call JavaScript from GDScript
JavaScriptBridge.eval("alert('Hello from Godot!');")

# Call a JavaScript function and get the return value
var result = JavaScriptBridge.call("myFunction", [1, 2, 3])
```

JavaScript can call Godot functions:

```javascript
// Assuming `godot` is the exported module instance
godot.call("my_gdscript_method", [arg1, arg2]);
```

## 6.  The HTML5 shell class

`HTML5Shell` is a singleton that handles browser integration:

- Fullscreen toggling
- Window resize events
- Orientation changes

You can extend it in GDScript:

```gdscript
class_name MyShell
extends Node

func _ready():
    var shell = HTML5Shell.get_singleton()
    shell.set_fullscreen(true)
```
