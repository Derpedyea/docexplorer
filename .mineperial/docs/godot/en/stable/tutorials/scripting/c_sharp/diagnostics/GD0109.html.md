**GD0109** – *The `[ExportToolButton]` attribute cannot be used with another `[Export]` attribute*  

---

### Rule information
| | |
|---|---|
| **Rule ID** | GD0109 |
| **Category** | Usage |
| **Fix is breaking or non‑breaking** | Non‑breaking |
| **Enabled by default** | Yes |

---

### Description  
In C# scripts, a property cannot simultaneously carry both the `[ExportToolButton]` attribute (used to expose a tool button in the editor) **and** a standard `[Export]` attribute.  

---

### Cause  
A property is annotated with both attributes, e.g.:

```csharp
[Export]
[ExportToolButton]
public int MyProperty { get; set; }
```

or

```csharp
[ExportToolButton]
[Export]
public string MyString { get; set; }
```

The compiler emits **GD0109** because the two attributes conflict: `[ExportToolButton]` is only intended for methods that return a `Callable`, not for regular exported properties.

---

### Fix  
Remove one of the conflicting attributes so that the property is either:

```csharp
[Export]
public int MyProperty { get; set; }
```

or

```csharp
[ExportToolButton]
public void MyButton() { ... }
```

If a tool button is needed, keep the `[ExportToolButton]` on a method and **do not** combine it with `[Export]` on the same member.

---

### Related rules  
- **GD0110** – The exported tool button is not a `Callable`  
- **GD0108** – The exported tool button is not in a tool class

---

### Example  
```csharp
// ❌ This will trigger GD0109
[Export]
[ExportToolButton]
public int Health { get; set; }

// ✅ Correct usage
[Export]
public int Health { get; set; }

// ✅ Tool button only
[ExportToolButton]
public void ResetHealth() {
    Health = 100;
}
```
