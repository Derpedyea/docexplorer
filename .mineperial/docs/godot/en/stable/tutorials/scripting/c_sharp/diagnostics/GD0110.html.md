**GD0110: The exported tool button is not a Callable**  
*Godot Engine – C# diagnostics*

---

### Overview
`GD0110` is a diagnostic rule that triggers when a property is exported with the `[ExportToolButton]` attribute but its type is **not** `Callable`. This prevents the editor from correctly exposing the tool button functionality.

---

### Rule Information

| Field | Value |
|-------|-------|
| **Rule ID** | `GD0110` |
| **Category** | Usage |
| **Fix is breaking or non‑breaking** | **Breaking** – change the property’s type to `Callable`. **Non‑breaking** – replace the `[ExportToolButton]` attribute with `[Export]`. |

---

### Quick Fix Examples

```csharp
// ❌ This will trigger GD0110
[ExportToolButton]
public int SomeProperty;

// ✅ Fix 1 – use a Callable
[ExportToolButton]
public Callable SomeProperty;

// ✅ Fix 2 – change to a normal exported property
[Export]
public int SomeProperty;
```

---

### Related Diagnostics

| Rule | Description |
|------|-------------|
| **GD0109** | The `[ExportToolButton]` attribute cannot be used with another `[Export]` attribute. |
| **GD0111** | The exported tool button must be an expression‑bodied property. |

---

### How to Resolve

1. **Identify the property** that has the `[ExportToolButton]` attribute and a non‑`Callable` type.  
2. **Decide on the intended use**:
   * If you want a tool button that calls a method, change the property to `Callable`.
   * If a simple exported value is desired, replace `[ExportToolButton]` with `[Export]`.
3. **Rebuild** the project to clear the diagnostic.

---

### TL;DR

- **GD0110**: `ExportToolButton` must target a `Callable`.
- **Breaking** – change type to `Callable`.
- **Non‑breaking** – switch to a normal `[Export]` attribute.
