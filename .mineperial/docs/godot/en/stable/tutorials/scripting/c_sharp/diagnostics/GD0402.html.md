**__GD0402: The class must not be generic__**  
*Godot Engine – C# diagnostics – stable documentation*

---

## Rule ID
`GD0402`

## Category
`Usage`

## Fix is breaking or non‑breaking
`Breaking`

## Enabled by default
Yes

---

## Cause
A **generic type** is annotated with the `[GlobalClass]` attribute.

> Godot’s GDScript integration expects a concrete, non‑generic type for classes that are exposed to the engine.  
> Generics are not supported in the runtime binding, so such classes cannot be instantiated or used from GDScript or the editor.

---

## Description
This rule flags any C# class that

```csharp
[GlobalClass]
public class MyGenericClass<T> : GodotObject
{
}
```

where `MyGenericClass` is generic.  
The compiler can’t expose such a class to the engine, leading to runtime failures or incomplete bindings.

---

## Recommendation
* Remove the `[GlobalClass]` attribute **or**
* Make the class non‑generic (i.e. drop the type parameter).

If you need generic behaviour, expose a concrete wrapper or use a non‑generic helper that forwards to generic logic.

---

## Example

### Invalid (will trigger GD0402)

```csharp
[GlobalClass]
public partial class DataHolder<T> : Node
{
    public T Value { get; set; }
}
```

### Valid

```csharp
public partial class IntHolder : Node
{
    public int Value { get; set; }
}
```

or

```csharp
public partial class DataHolder : Node
{
    public object Value { get; set; }
}
```
